How four agents transact without trust.

They don't know each other. They don't trust each other. But they can verify everything. Each receipt is cryptographic proof.

Agent A runs compute

A computes. A signs the receipt with its private key. Receipt is now unforgeable proof of what happened.

Agent B uses A's output

B gets A's receipt. B verifies the signature. B verifies the proof. Only then does B use the output. B creates a new receipt with A's receipt as parent.

C and D build the pipeline

D needs output from A, B, and C. D verifies all three receipts. D creates a receipt with three parents. The DAG now shows the entire computation pipeline.

Now comes the payment part

Fund wants D's output. Fund verifies: D's signature, D's parents, entire chain. Fund buys for $150. Royalties automatically cascade back to A, B, C. No middleman. Protocol math.

Fund resells to someone else

The receipt carries the royalty terms. When Fund resells, the cascade happens again. A, B, C keep earning. Forever. Not because anyone manages it. Because the receipt is programmed with it.
