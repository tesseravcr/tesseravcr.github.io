Tessera VCR
Explorer Demo Spec Source

Run the spec locally.

Four agents. Each has something the others lack. They transact through verified receipts, not trust.

Every signature is real Ed25519. Every hash is real SHA-256. The TEE attestation is a real hardware-signed document from an AWS Nitro Enclave.

This is what the protocol looks like in practice.

Setup

git clone https://github.com/tesseravcr/tesseravcr.git
cd tesseravcr
pip install -r tessera-py/requirements.txt
python3 tessera-py/demo.py

Takes ~30 seconds. Prints the entire protocol flow to stdout. No network calls. Everything local.

What you'll see

1. Four agents introduce themselves

Each generates an Ed25519 keypair. Each one is a provider with a different model.

2. Agent A asks Agent B for compute

B runs a model. Proves it with a Nitro TEE attestation. Returns the output + proof.

3. A creates a receipt

Receipt includes: what model ran. Who ran it. What went in. What came out. Proof of execution. A's signature. Everything.

4. A resells the output to C

C gets the receipt + proof. C can verify B actually ran the model. C can verify A didn't tamper with the output.

5. Royalties flow back

When A sells to C, B automatically gets a cut. Protocol handles it. No accounting. No middleman. Just math.

6. C resells to D

D verifies the entire provenance chain. A's signature. B's proof. C's transfer. All cryptographically locked together.

The output

You'll see hex dumps of every structure. Receipt IDs. Merkle trees. Ed25519 signatures. Hashes. All real.

Scroll through it. Read the comments. That's the spec in action.

Source code is 500 lines. Heavily commented. No magic.
